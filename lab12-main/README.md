# Аутентификация и работа с токенами (JWT, OAuth)

## Цель работы
Целью данной работы является изучение и внедрение механизма аутентификации с использованием токенов, таких как JWT (JSON Web Token). В рамках проекта реализована простая система входа с использованием фейковых JWT токенов для имитации аутентификации пользователей и хранения их сессий.

## Краткое описание программы
Программа представляет собой простую веб-страницу с формой входа, где пользователь вводит свою электронную почту и пароль. После успешной аутентификации генерируется фейковый JWT токен, который сохраняется в `localStorage` браузера. Страница поддерживает переключение между темной и светлой темами с помощью кнопки и отображает личные данные пользователя после входа.

- При успешном входе отображается страница с приветственным сообщением и кнопкой для выхода.
- При смене темы меняется внешний вид страницы.
- Вход осуществляется через сгенерированный фейковый JWT токен.

## Выводы по работе
В ходе работы была реализована базовая система аутентификации с использованием фейковых JWT токенов. Были реализованы следующие ключевые аспекты:
- Генерация и хранение JWT токенов в `localStorage`.
- Управление состоянием входа с проверкой на наличие действующего токена.
- Переключение между темной и светлой темой.
- Работа с элементами DOM для изменения состояния страницы в зависимости от аутентификации пользователя.

Это демонстрационный проект, и для реальной эксплуатации требуется интеграция с настоящим сервером для работы с безопасными токенами и реальной аутентификацией.
