## Описание проекта — “Менеджер расходов”
В рамках лабораторной я разработал небольшое веб-приложение для учёта расходов.
Оно позволяет фиксировать траты, указывать сумму и категорию, хранить данные в LocalStorage и быстро находить нужные записи через поиск.
Написан на чистом html, js и стили на tailwind что влияет на высокую скорость быстродействия.

Основные возможности приложения:

- добавление нового расхода;
- удаление элемента;
- поиск по категории;
- хранение данных в `Map`;
- автоматическое сохранение в LocalStorage;
- работа с DOM через отдельный ui-модуль;
- модульная структура проекта;
- один unit-тест (Jest);
- возможность технической отладки через DevTools.

Приложение может использоваться как простейший личный бюджет-трекер.

---

### Техническое описание модулей

### 1. `storage.js`
Отвечает за сохранение и загрузку данных через LocalStorage.
Для хранения используется структура `Map`, что удобно при работе с уникальными id.

### 2. `ui.js`
Содержит функцию `render()`, отвечающую за отображение списка расходов на экране.

### 3. `utils.js`
Вспомогательные функции: генерация id и тестируемая функция `sum()`.

### 4. `main.js`
Логика приложения: обработка событий, поиск, добавление, удаление, рендер.
